# Claims and Non-Claims

## Package

Fork Executive Buyer Packet v0.1

## Supported claims

This package supports the following claims:

1. Fork addresses a reconstructive-fidelity gap in AI-assisted workflows.
2. Fork is positioned as a read-only, out-of-band, recomputable evidence layer.
3. Fork preserves bounded evidence records for later review.
4. Fork helps legal, compliance, audit, risk, security, governance, and operations functions inspect what was preserved and what cannot be claimed.
5. Fork is designed to integrate beside existing institutional systems rather than replace them.
6. Fork's commercial path begins with bounded workflow evaluation and pilot discovery.

## Non-claims

This package does not claim:

1. Fork is production-deployed in any enterprise environment.
2. Fork is ready for a specific client workflow.
3. Fork guarantees source completeness.
4. Fork proves AI output correctness.
5. Fork proves decision correctness.
6. Fork asserts legal admissibility.
7. Fork satisfies compliance obligations.
8. Fork performs audit.
9. Fork performs legal review.
10. Fork owns remediation.
11. Fork owns reporting.
12. Fork blocks, routes, approves, denies, or modifies live workflows.
13. Fork can reconstruct hidden vendor behavior.
14. Fork can recover evidence that was never observed, exported, attested, captured, or referenced.
15. Fork replaces governance, GRC, workflow, legal operations, audit, security, or AI infrastructure systems.

## Boundary statement

Fork is the evidence hand-off.

It preserves bounded evidence records for later inspection.

It does not become the upstream authority.

It does not become the downstream judgment.

It does not become the control plane.
