# Next Steps

## For executive buyers

Review:

1. Executive brief
2. Buyer problem
3. Stakeholder map
4. Use cases
5. Pilot options
6. Questions for buyers
7. Claims and non-claims

Then determine whether the next step should be:

- technical validation review
- pilot discovery
- internal stakeholder mapping
- no further action

## For technical reviewers

Use the Technical Validation Packet when the question becomes inspectable mechanics, package validation, schemas, examples, tests, tools, or verifier behavior.

## For pilot prospects

Use the Pilot Discovery Packet when the question becomes workflow suitability, source-system inventory, evidence boundary mapping, and pilot scoping.

## For implementation discussions

Do not proceed directly from this packet to implementation.

A pilot-ready implementation packet requires prior discovery, source-system review, security and data-handling review, workflow scope, acceptance criteria, and client-specific non-claims.
