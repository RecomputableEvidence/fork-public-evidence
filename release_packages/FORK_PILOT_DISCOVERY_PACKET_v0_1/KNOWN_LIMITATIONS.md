# Known Limitations

## Purpose

This document identifies known limitations of the Pilot Discovery Packet.

## Limitations

This package is a discovery scaffold.

It does not prove that a workflow is suitable.

It does not define a client-specific evidence boundary.

It does not establish production readiness.

It does not include client source-system analysis.

It does not include security approval.

It does not include legal approval.

It does not include commercial terms.

It does not include deployment instructions.

It does not include client-specific acceptance criteria.

## Technical limitations

Pilot discovery may reveal that:

- Source systems cannot export relevant evidence.
- Relevant evidence is overwritten or expired.
- Vendor telemetry is incomplete.
- AI model behavior cannot be replayed.
- Prompt or retrieval context was not retained.
- Human review was not captured.
- Approval and execution states are collapsed in existing systems.
- Policy snapshots are unavailable.
- Source completeness cannot be responsibly claimed.

## Boundary limitation

Fork can only preserve what is observable, captured, referenced, or explicitly declared unavailable within the defined boundary.

Fork should not invent evidence to close a gap.
