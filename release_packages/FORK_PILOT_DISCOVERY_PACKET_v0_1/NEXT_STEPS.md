# Next Steps

## For a serious prospect

Start with:

1. Candidate workflow identification
2. Stakeholder identification
3. Source-system inventory
4. Evidence boundary worksheet
5. Security and data-handling questions
6. Pilot suitability decision

## For Fork

Use discovery to determine whether to prepare:

- Pilot-Ready Implementation Packet
- Client Evidence Boundary Packet
- Technical Validation Packet addendum
- Executive Buyer Packet addendum

## For technical reviewers

Use the Technical Validation Packet to inspect public technical mechanics before pilot scoping.

## For executive reviewers

Use the Executive Buyer Packet when the question is budget, sponsorship, risk ownership, or commercial fit.

## For client-specific work

Do not proceed to implementation without a client-specific evidence boundary.

Discovery must produce a bounded workflow and explicit non-claims before any pilot-ready package is prepared.
