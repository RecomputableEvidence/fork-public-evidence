# Claims and Non-Claims

## Package

Fork Public Doctrine Packet v0.1

## Supported claims

This package supports the following claims:

1. Fork has a public doctrine posture.
2. Fork is framed as a recomputable evidence layer for AI-assisted workflows.
3. Fork is bounded as read-only and out-of-band.
4. Fork preserves evidence of hand-offs rather than absorbing upstream or downstream institutional functions.
5. Fork emphasizes evidence integrity rather than AI trust.
6. Fork distinguishes evidence preservation from authority, control, compliance, audit, legal judgment, remediation, and reporting.
7. Fork public materials include doctrine, boundary, and executable evidence constraints in the public repository.

## Non-claims

This package does not claim:

1. Fork is production-deployed in any enterprise environment.
2. Fork is pilot-ready for any specific client workflow.
3. Fork guarantees source completeness.
4. Fork proves AI output correctness.
5. Fork proves decision correctness.
6. Fork asserts legal admissibility.
7. Fork satisfies compliance obligations.
8. Fork validates authority as legally sufficient.
9. Fork performs audit.
10. Fork performs legal review.
11. Fork owns remediation.
12. Fork owns reporting.
13. Fork blocks, routes, approves, denies, or modifies live workflows.
14. Fork can reconstruct hidden vendor behavior that was never observed, exported, attested, or captured.
15. Fork can recover evidence that was never captured or referenced.

## Boundary statement

Fork preserves the evidence boundary.

It does not become the upstream authority.

It does not become the downstream judgment.

It remains the evidence hand-off.
