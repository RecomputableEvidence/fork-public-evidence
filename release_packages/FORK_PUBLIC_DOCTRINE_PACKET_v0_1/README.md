# Fork Public Doctrine Packet v0.1

## Purpose

This package provides the public doctrine orientation packet for Fork.

It is intended for early reviewers, AI governance practitioners, legal, audit, compliance, risk, security, and technical readers who need to understand what Fork is, what Fork is not, and where the public evidence posture currently stands.

This package is not a pilot implementation package.

This package is not a client-specific evidence boundary packet.

This package is not a production deployment claim.

## Package role

The Public Doctrine Packet establishes the public category and boundary for Fork:

Fork is a read-only, out-of-band, recomputable evidence layer for AI-assisted workflows.

Fork preserves bounded evidence records so later reviewers can inspect what was observable and captured.

Fork does not decide whether an action was valid.

Fork does not prove the AI output was correct.

Fork does not become the authority, control plane, compliance engine, audit function, legal reviewer, remediation owner, or reporting function.
