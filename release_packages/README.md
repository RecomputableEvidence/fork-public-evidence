# Release Packages Boundary

## Status

This directory contains Fork public-review package materials.
The phrase `release_packages` means packages released for bounded public review.
It does *not* mean product release, production deployment, customer deployment, commercial availability, procurement readiness, or certification.

## What these packages may support

A package in this directory may support bounded review of:

- declared public doctrine
- package membership
- checksum integrity
- technical validation materials
- executive or buyer orientation
- pilot-discovery scoping
- claims and non-claims declared by the specific package

## What these packages do not establish

Packages in this directory do not establish:

- legal admissibility
- production readiness
- customer deployment
- commercial pilot approval
- pricing commitment
- procurement readiness
- SOC 2, ISO, HIPAA, or regulatory compliance
- audit sufficiency
- risk acceptance
- security certification
- source completeness
- AI-output correctness
- workflow correctness
- institutional authority

## Correct review path

Start with:

- `REVIEWER_QUICK_START_v0_1.md`
- `docs/REVIEWER_START_HERE_v0_1.md`
- `docs/PUBLIC_REVIEW_PACKAGE_INDEX_v0_1.md`
- `docs/FORK_NON_CLAIM_BOUNDARY_v0_1.md`

Then inspect only the package that matches the review question.

## Package interpretation rule

Each package is bounded by its own `README.md`, `CLAIMS_AND_NON_CLAIMS.md` where present, and the repository-wide non-claim boundary.

Do not infer a broader claim from a package name, directory name, or successful checksum/package verification.
