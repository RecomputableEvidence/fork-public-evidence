# Outreach Message Variants v0.1

## Purpose

This document defines Fork's outbound message families for selective outreach.

Fork outreach preserves three separate paths:

1. Public reviewer
2. Advisor / design partner
3. Enterprise bounded workflow PoV

Each path has a different audience, asset anchor, tone, and ask.

## Boundary

These messages are public-safe templates.

They are not endorsements, customer claims, production-readiness claims, legal opinions, compliance claims, security approvals, procurement offers, binding quotes, or evidence that any recipient has reviewed or accepted Fork.

Do not commit personalized messages, target names, private relationship notes, or reply content to the public repository.

## Message spine

Every message should:

1. Establish category clarity.
2. Identify the specific problem.
3. Point to artifact-backed credibility.
4. Preserve the non-claim boundary.
5. Offer one low-risk next step.

## Do not collapse the paths

Do not send the enterprise PoV message to public reviewers.

Do not send the public reviewer message to buyers.

Do not send the advisor/design-partner message to someone who should be asked for a scoped enterprise conversation.
