# Longitudinal Recomputation v0.2 — Non-Claims and Limits

This candidate deepens Fork from declared temporal reconciliation toward
evidence-derived replay across an exact Git interval. It remains a research
candidate and is not an admitted modular surface.

## What the candidate can establish

- every Git commit in the declared ancestry-path interval is classified;
- every registered event binds its exact commit, tree, parents, changed paths,
  and evidence bytes;
- the standing vector can be regenerated from the replay-start state,
  normalized events, and a consumer-owned Sequence Surface reducer;
- each changed and preserved standing dimension is reported separately;
- review standing remains attached to the exact reviewed head;
- committed projections and transition receipts match deterministic replay.

## What it cannot establish

- that events outside the declared interval were discovered;
- that every semantic event inside repository history was classified correctly;
- truth, correctness, causality, safety, compliance, or legal sufficiency;
- reviewer identity or institutional authority beyond the bound receipt;
- current mutable GitHub pull-request metadata without a separate captured
  metadata receipt;
- admission, publication, merge authority, provider authority, retry authority,
  readiness, or execution permission.

Candidate discovery may be automated. Event classification, admission, and
authority remain governed acts.
