# Event Runbook v0.1

Identifier: EVENT_RUNBOOK_v0_1
Status: Draft

## Phase 0: Repository gate

Before inviting broader observation:

1. Confirm clean branch.
2. Confirm no encoding corruption.
3. Confirm exterior-observations surface is navigable.
4. Confirm sandbox v0.2 question is visible.
5. Confirm observation intake template exists.
6. Confirm non-endorsement language is present.

## Phase 1: Internal LLM team observance

Ask:

Does the exterior-observations surface preserve scope, ambiguity, limitation, non-endorsement, and architectural evolution without converting exterior commentary into authority?

Collect:

- unclear language;
- implied consensus risks;
- navigation issues;
- overbroad claims;
- missing boundary statements.

## Phase 2: Controlled exterior observers

Invite a small bounded group across disciplines.

Suggested observer categories:

- governance;
- audit;
- procurement or ATO;
- systems architecture;
- compliance;
- AI assurance;
- adjacent governance-system builders.

Collect observations using the intake template.

## Phase 3: Coding and synthesis

Code observations using CODING_GUIDE_v0_1.

Update synthesis only as descriptive analysis.

Do not claim consensus, validation, or endorsement.

## Phase 4: Sandbox refinement

Promote recurring pressure cases into sandbox fixture candidates.

Prioritize cases that test distinguishability between:

- structural reproduction;
- unresolved state;
- evidentiary sufficiency;
- authority;
- truth-claims.

## Phase 5: Publication gate

Before broader publication, confirm:

- all preserved observations have attribution status;
- no observation is overstated;
- no observer is represented as authority;
- no claim exceeds the evidence surface;
- every pressure case is marked by maturity status.
