# AHI Scenario Ladder v0.3

## Purpose

The AHI scenario ladder is a progressive boundary-failure casebook.

Each scenario isolates a different way an AI-assisted artifact, claim, state, authority context, or evidence basis can be silently promoted during handoff.

## Ladder

### Scenarios 01–02 — Baseline handoff contrast

| Scenario | Boundary lesson |
|---:|---|
| 01 | An unbounded handoff allows downstream reliance to infer more than the artifact supports. |
| 02 | Fork preserves the handoff boundary so a reviewer can inspect what was and was not carried forward. |

### Scenarios 03–05 — Claim, authority, and policy laundering

| Scenario | Boundary lesson |
|---:|---|
| 03 | A bounded claim can be expanded downstream unless unsupported scope expansion remains visible. |
| 04 | Authority context does not transfer merely because an artifact crosses systems. |
| 05 | Policy references and suppressed non-claims can be laundered into apparent sufficiency without explicit boundary preservation. |

### Scenarios 06–07 — Multi-system and external authority boundaries

| Scenario | Boundary lesson |
|---:|---|
| 06 | Distributed systems can each remain internally valid while the cross-system transition fails. |
| 07 | External review context can be referenced without becoming inherited authority. |

### Scenarios 08–09 — Temporal validity and split-state propagation

| Scenario | Boundary lesson |
|---:|---|
| 08 | Prior validity, authority, or evidence sufficiency does not imply current validity, authority, or evidence sufficiency. |
| 09 | A current validity-changing event does not automatically become visible, consumed, or operative across downstream systems. |

## Current edge

```text
Scenario 08: prior validity does not imply current validity
Scenario 09: current invalidation does not automatically propagate
```

A natural next scenario would test acknowledgment, synchronization, and reconciliation boundaries.
