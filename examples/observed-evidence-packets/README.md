# Observed Evidence Packet examples

This directory contains an educational two-case suite demonstrating Fork recomputability-class boundaries.

## Executable examples

- `case-A-hosted` — hosted LLM output classified as `NON_RECOMPUTABLE`.
- `case-C-strong` — deterministic local generation classified as `STRONG_RECOMPUTATION` for `artifacts/brief.txt`.

These examples are educational only. They must not be merged as policy without governance review.

## Boundary rules

- `manifest.json` contains canonical packet facts only.
- `manifest.json` must not contain `manifest_digest`, `signatures`, or `gate_receipts`.
- `SHA256SUMS` stores the canonical manifest digest and artifact digests.
- `receipt.template.json` is illustrative only. Do not treat it as a real receipt.
- Real `receipt.json` files must reference `sha256:<manifest_digest>` after the digest is computed.
- `verification_result.json` is generated by the verifier and is not canonical packet input.

## Signature status

Cryptographic signature verification is `NOT_CHECKED` in this educational suite. The verifier checks receipt digest references only when a real `receipt.json` is present. A production extension may add GPG or OpenSSL verification with public keys under `keys/`.

## Experimental concepts

`PARTIAL_RECOMPUTATION` and `PRESERVED_ONLY` are experimental discussion concepts only in this suite. They are not schema-valid executable classes here.
