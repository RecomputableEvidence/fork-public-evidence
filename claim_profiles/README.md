# Claim Profiles Boundary

## Status

This directory contains Fork claim-boundary profiles and evidence-scope profile material.

These files are not compliance-control templates, GRC control assertions, regulatory mappings, audit findings, certification statements, or customer-specific claim packs.

## Correct interpretation

A claim profile may describe the bounded evidentiary scope under which a Fork artifact, receipt, checker, or package should be interpreted.

A claim profile does not establish that the underlying workflow, organization, model, vendor, control, policy, or decision is:

- compliant;
- safe;
- lawful;
- complete;
- audited;
- certified;
- approved;
- production-ready;
- independently validated;
- suitable for a customer environment.

## GRC interpretation boundary

Do not treat this directory as a library of compliance claims.

Do not map a claim profile directly to:

- control satisfaction;
- audit sufficiency;
- legal admissibility;
- regulatory compliance;
- risk acceptance;
- remediation closure;
- customer approval;
- production authorization.

Any such mapping requires a separate authorized governance, legal, audit, compliance, or risk process with its own evidence and accountability.

## Citation rule

When citing a claim profile, cite the specific file and state its declared evidence boundary.

Do not cite the `claim_profiles/` directory as a general assertion that Fork establishes compliance, control effectiveness, audit readiness, or institutional authority.
